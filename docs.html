Arjan Translate

Automatic localization + Translation of HTML files using neural machine translation

You can use Arjan in three ways
  1. The Arjan translation GUI:
    1. Pros: GUI
    2. cons:
      1. only translate a single page at a time
      2. cont update translations
  2. Programmatic usage
    1. Pros: integrate into other programs and workflows
  3. Arjan CLI (coming soon)
    1. pros:
      1. translate multiple pages
      2. bi-directional translation updates
    2. cons: No GUI. basic terminal usage knowledge

Installation

npm i arjan-translate

Usage

  1. go into your sites directory cd SITE_NAME
  2. run arjan init SITE_NAME Refer to the provider setup section if you haven't used any of the cloud translation APIs.
  3. Run the translate command arjan translate SITE_NAME [FILENAME]

Then what?

the translate command generates 3 things (or maybe 4)

  1. locale JSON files for the input and output languages
  2. translated html file/s with the output language code (es.html or es/about.html)
  3. localized html files. i18n compatible
  4. CSV file with translations

Once you have translated your doc you can improve all of your translations by working directly on your neatly organized JSON locale files and running the translate command with the —update flag (-u).

Arjan translations is bi-directional meaning that you can also work on the output HTML files and then run the translate command with the —backwards (-b) flag to update your JSON files.

You can also generate a single CSV file with all the translations for your site by running the translate command with the —export flag (-e). if you use both the —export and —backwards flags you can update the CSV with site data. if you provide the filename arg only translations for that file will be included in the CSV.

Methods

createLocale(html, filename, from, to, callback):
  1. Parses the HTML of the input file;
  2. for all elements that contain text it adds an ID
  3. generates or re-writes a JSON file that contains ids and the text values.
traslateLocale(input, filename, from, to, size, callback):
  1. Translates each value using amazon translate
  2. generates or re-writes a JSON file that contains ids and translated values
translateHtml(filename, html, translation, translations, to, callback):
  1. copies the input HTML
  2. replaces all values with the translated value
translateSite(from, to):
  1. createLocale
  2. translateLocale
  3. translateHtml
exportCsv(lang, filename):
  1. Exports a desired JSON locale as a CSV
importCsv(lang, filepath):
  1. Updates a JSON locale with data from a CSV file

CLI Commands (coming soon)

init

arjan init SITE_NAME [PROVIDER] Args
  • Site Name: required → String → The name of your site e.g. amazon or amazon.com. Include the TLD (.com) if you have another project with the same name and a different TLD like .org or something.
  • Provider:optional → aws|gcp → The desired translation provider. You can choose google cloud or AWS
Flags
  • profile: local AWS profile name
  • region: AWS region e.g. us-east-1

translate

arjan translate SITE_NAME \[FROM_LANG\] [TO_LANG] [FILENAME]Args
  • Site Name → required → the name of your site e.g. amazon or amazon.com. Include the TLD (.com) if you have another project with the same name and a different TLD like .org or something.
  • From Lang → required → origin language code ex. en
  • To Lang → optional → destination language code ex. es. If the argument is not provided or the same origin lang is used only the origin site will be procesed and parsed.
  • File Name → optional → the name of your file e.g. index or index.html not necessary to include the .html. if filename is not provided, it will translate all of the html files inside the directory. For quicker builds in large websites its recommended to use this arg if you only change a few files.
Flags
  • new → true|false → Generate a new translation
  • update → true|false → Update your translated HTML files according to changes in your JSON locales.
  • change → true|false → If you make changes to one of your files (e.g. added a new section or changed some of the wording, and you want to reflect these changes in your other languages you can use the —change (-c) flag. (this is the most memory intensive operation)
  • backwards → true|false → Update your JSON locales according to changes in your HTML files.
  • format → arjan|i18n|both → Choose your desired HTML file build format. arjan, i18n or both (arjan is default). When building backwards its important to provide this flag if you want to use i18n.
  • export → true|false → Produce a CSV file with all of your translation data
  • html5 → true|false → produce parent objects in JSON schema.

Translation Format

You can translate a static site into one of two output formats:

  1. Arjan: Great for static websites
  2. i18n: Widely accepted and supported translation format for apps.

Lets suppose our input is an en.html file with the following content: 

<section>
  <h1 id="title1">Arjan is super cool</h1>
</section>
After running the translate command we would get the following output:
  1. locales/en.json → { "title1":"Arjan is super cool" }
  2. en.html
    1. Arjan → <h1 id="title1"> Arjan is super cool </h1>
    2. I18n → <h1 id="title1"> {{arjan.t('title1)}} </h1>
Lets suppose that our input string didnt have an id attribute:
  1. locales/en.json → { "arjan_is_sup1":"Arjan is super cool" }
  • en.html
    1. Arjan → <h1 id="arjan_is_sup1"> Arjan is super cool </h1>
    2. en.html → <h1> {{arjan.t('arjan_is_sup1')}} </h1>

    Notice that an id with the first 12 characters of the string is created. Caps are lower-cased and spaces are replaced with underscores. A number with the index of the translations is inserted at the end (in case there’s another string that starts with the same 12 chars)

    HTML5 (Coming Soon)

    • if you are using html5 elements in your page (nav, header, section, footer) you can add the html5 option in the translate command. This will generate objects with ids of html5 elements (nav, header, section, footer) and will insert translations as children of the object they belong to. suppose were still using the example above without ids:
      1. translations/en.json → { "section1":{"arjan_is_sup":"Arjan is super cool" }}
    • en.html
      1. Arjan → <section id="section1"><h1 id="section1_arjan_is_sup"> Arjan is super cool </h1></section>
      2. I18n → <section><h1> {{arjan.t('section1.arjan_is_sup')}} </h1></section>

      Dont translate (Coming Soon)

      You can use the HTML translate tag to tell arjan if you dont want to translate a particular element by setting translate="no" in your element

    Provider Setup

    before doing any of this you must have an account in Amazon Web Services or an account in Google Cloud Platform with a billing plan. Creating an account in either platform is easy and free. and unless your are translating more than 20 million characters per month both translation services are free.

    AWS

    1. Run arjan init SITE_NAME aws -r=your-aws-region -p=profile-name from the root of your project's directory
  • Finish creating your IAM user in the AWS Console and hold on to the access keys.
  • Update the local profile in your machine. The local profiles are stored in ~/.aws/credentials in mac/linux or in C:\Users\USER_NAME\.aws\credentials in windows. you can create/edit this file by runing sudo nano ~/.aws/credentials. add the profile keys in the format shown bellow.
    •  [profilename]
 aws_access_key_id = <YOUR_ACCESS_KEY_ID>
 aws_secret_access_key = <YOUR_SECRET_ACCESS_KEY>

    Google Cloud (Coming Soon)

    1. Go to https://cloud.google.com/translate/docs/basic/setup-basic and click on the set up a project button.
    2. Click to:
      3. - Create a project, you can call it arjan translate if you like, then click next
- Enable the Cloud Translation API for that project.
- Create a service account.
- Download a private key as JSON.
    3. Set the environment variable GOOGLE_APPLICATION_CREDENTIALS to your service account key.

    Then run the init command arjan init SITE_NAME gcs

    Programmatic Use

    Installation

    npm i arjan_translate

    Usage

    const arjanTranslate = require('arjan-translate')

//REGION is the AWS region of your IAM role ex. 'use-east-1'
//PROFILE is the name of your desired AWS IAM profile ex. 'deafult'

arjanTranslate.Build('REGION', 'PROFILE', function(err, data){
  if(err) console.log(err)
  else {
    //FROM is the language code of your origin file ex. 'en'
    //TO is the language code of your destination file ex. 'es'
    arjanTranslate.TranslateSite('FROM', 'TO');
  }
})

    Arjan translate GUI

    Arjan translate also has a GUI at arjan.tools/translate

    the GUI is pretty limited as you cant update your translations but its good for a one time job especially if you have no programming experience. Its also a cool way of trying out the tool and checking out the format.

    The GUI is a form with a from field, a to field, and a drop zone were you can drag and drop your files. Only HTML, txt, and markdown files supported.
